# ☀️ Sistema de Business Intelligence: Monitoreo de Radiación UV (AEMET)

Este proyecto ha sido desarrollado para la asignatura de **Complementos de Bases de Datos**. El objetivo es el diseño e implementación de un almacén de datos (Data Warehouse) que centralice y analice los niveles de radiación ultravioleta en España utilizando datos oficiales de la **AEMET**.

## 👥 Autoras
* **Olga Cantalejo Gómez**
* **Paula María Suárez Linares**

## 🚀 Descripción del Proyecto
El sistema transforma datos meteorológicos en bruto (formato CSV) en un modelo relacional multidimensional optimizado para la toma de decisiones y el análisis de salud pública. 

### Stack Tecnológico
* **Base de Datos:** Oracle Database 21c Express Edition (XE).
* **ETL:** Pentaho Data Integration (Spoon) v11.
* **Lenguaje:** SQL (PL/SQL).

## 📊 Arquitectura de Datos
Se ha implementado un **Modelo en Estrella**, seleccionado por su alto rendimiento en consultas de agregación y su simplicidad para el análisis de BI.

* **Tabla de Hechos (`FACT_RADIACION`):** Almacena el índice UVI máximo diario.
* **Dimensión Tiempo (`DIM_TIEMPO`):** Desglose temporal de las mediciones.
* **Dimensión Ciudad (`DIM_CIUDAD`):** Información geográfica de las estaciones de la AEMET.

## 🛠️ Instalación y Despliegue

### Requisitos Previos
1.  **Java JDK 8 u 11** instalado.
2.  **Oracle Database 21c XE** en funcionamiento.
3.  **Driver JDBC de Oracle** (`ojdbc8.jar`) copiado en la carpeta `data-integration/lib`.

### Pasos para ejecutar
1.  **Base de Datos (Configuración del esquema):** Ejecuta el siguiente bloque como usuario SYSTEM para preparar el entorno:
    ```sql
    -- 1. Entrar en el contenedor correcto (Oracle 21c XE)
    ALTER SESSION SET CONTAINER = XEPDB1;

    -- 2. Crear el usuario del proyecto
    CREATE USER METEO_ESTRELLA IDENTIFIED BY "tu_contraseña";

    -- 3. Asignar permisos y cuota de almacenamiento
    GRANT CONNECT, RESOURCE, DBA TO METEO_ESTRELLA;
    ALTER USER METEO_ESTRELLA QUOTA UNLIMITED ON USERS;

    -- 4. Ejecutar los scripts de las tablas en este orden:
    --    01_DIM_CIUDAD.sql -> 02_DIM_TIEMPO.sql -> 03_FACT_RADIACION.sql
    ```
2.  **Configuración ETL:** Abre Pentaho Spoon y carga el archivo `.ktr`.
3.  **Conexión:** Ajusta la conexión de base de datos con los siguientes parámetros:
    * **Host:** localhost
    * **Puerto:** 1521
    * **Service Name:** /XEPDB1
4.  **Ejecución:** Pulsa el botón "Run" en Pentaho para iniciar la carga de las 59 filas de datos.

## ⚠️ Retos Técnicos Superados
* **Error ORA-12505 (Conectividad)**: Resolución de conflictos de conexión mediante la transición de configuración por SID a Service Name (/XEPDB1), necesaria para la arquitectura de Oracle 21c XE.
* **Limpieza de Datos:** Migración del componente CSV file input al componente Text file input para gestionar correctamente los metadatos (cabeceras y pies de página) del fichero de la AEMET.
* **Integridad:** Uso de `GENERATED BY DEFAULT AS IDENTITY` para la gestión automática de claves subrogadas.
