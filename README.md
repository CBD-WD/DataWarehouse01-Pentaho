# ☀️ Sistema de Business Intelligence: Monitoreo de Radiación UV (AEMET)

Este proyecto ha sido desarrollado para la asignatura de **Componentes de Bases de Datos**. El objetivo es el diseño e implementación de un almacén de datos (Data Warehouse) que centralice y analice los niveles de radiación ultravioleta en España utilizando datos oficiales de la **AEMET**.

* **Olga Cantalejo Gómez**
* **Paula María Suárez Linares**

## 🚀 Descripción del Proyecto
El sistema transforma datos meteorológicos en bruto (formato CSV) en un modelo relacional multidimensional optimizado para la toma de decisiones y el análisis de salud pública. 

### Stack Tecnológico
* **Base de Datos:** Oracle Database 21c Express Edition (XE).
* **ETL:** Pentaho Data Integration (Spoon) v11.
* **Lenguaje:** SQL (PL/SQL).

## 📊 Arquitectura de Datos
Se ha implementado un **Modelo en Estrella**, seleccionado por su alto rendimiento en consultas de agregación y su simplicidad para el análisis de BI.

* **Tabla de Hechos (`FACT_RADIACION`):** Almacena el índice UVI máximo diario.
* **Dimensión Tiempo (`DIM_TIEMPO`):** Desglose temporal de las mediciones.
* **Dimensión Ciudad (`DIM_CIUDAD`):** Información geográfica de las estaciones de la AEMET.

## 🛠️ Instalación y Despliegue

### Requisitos Previos
1.  **Java JDK 8 u 11** instalado.
2.  **Oracle Database 21c XE** en funcionamiento.
3.  **Driver JDBC de Oracle** (`ojdbc8.jar`) copiado en la carpeta `data-integration/lib`.

### Pasos para ejecutar
1.  **Base de Datos:** Ejecuta el script SQL en Oracle para crear el esquema y las tablas.
    ```sql
    -- Ejemplo de creación de esquema
    CREATE USER METEO_ESTRELLA IDENTIFIED BY contraseña;
    ```
2.  **Configuración ETL:** Abre Pentaho Spoon y carga el archivo `.ktr`.
3.  **Conexión:** Ajusta la conexión de base de datos con los siguientes parámetros:
    * **Host:** localhost
    * **Puerto:** 1521
    * **SID:** XE
4.  **Ejecución:** Pulsa el botón "Run" en Pentaho para iniciar la carga de las 59 filas de datos.

## ⚠️ Retos Técnicos Superados
* **Error ORA-12505:** Resolución de problemas de conexión mediante la configuración correcta del SID de Oracle.
* **Limpieza de Datos:** Tratamiento de cabeceras complejas y pies de página en el CSV de la AEMET mediante filtros de filas en Pentaho.
* **Integridad:** Uso de `GENERATED BY DEFAULT AS IDENTITY` para la gestión automática de claves subrogadas.
